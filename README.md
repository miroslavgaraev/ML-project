# Детектор Дипфейков

##  Описание проекта
Проект представляет собой систему автоматического обнаружения deepfake-изображений с использованием методов глубокого обучения. Модель принимает изображение лица на входе и возвращающает: **метку «Real / Fake»** и **две вероятности**. Система предназначена для повышения эффективности и точности идентификации поддельных фото в цифровой среде.

### Зачем нужна автоматическая система обнаружения дипфейков?
-	Ускорение модерации контента
-	Масштабируемость
-	Стандартизация
-	Предотвращение злоупотреблений и помощь в расследованиях

## Архитектура проекта
- **Архитектура нейронной сети:** Простая convolutional neural network (CNN) на Keras Sequential
- **Задача:** Бинарная классификация (real/fake)
- **Фреймворк:** TensorFlow/Keras
- **Размер входных изображений:** 128×128 пикселей

### Структура проекта
```
├── main.ipynb # Основной файл с кодом загрузки данных, обучения, оценки и предсказаний модели

```

## Ход работы

### Параметры обучения
- **Оптимизатор:** Adam (по умолчанию скорость обучения 0.001)
- **Функция потерь:** Categorical cross-entropy (для 2 классов)
- **Количество эпох:** 51 (с возможной ранней остановкой по валидации, если не указано)
- **Размер батча:** 32
- **Архитектура модели:** Sequential с Conv2D (32,64,128 фильтров), MaxPooling2D, Flatten, Dense(128), Dropout(0.5), Dense(2, softmax)

### Предобработка данных

- **Изменение размера:** Все изображения приводятся к размеру 128×128 пикселей
- **Цветовое пространство:** Конвертация в RGB формат (если необходимо)
- **Нормализация:** Стандартизация пикселей к диапазону [0,1] (делится на 255)
- **Аугментация** (только для тренировочных данных)
В ImageDataGenerator добавлены:
...повороты (rotation_range=20)
...ширина/высота сдвига (0.2)
...сдвиг (0.2)
...зум (0.2)
...горизонтальное отражение для увеличения разнообразия данных
- **Разделение данных**

### Разделение данных

- **Тренировочная выборка:** 70% данных (после аугментации)
- **Валидационная выборка:** 10% данных
- **Тестовая выборка:** 20% данных
- Используется библиотека split-folders для случайного разделения с seed=1337

## Результаты

**Accuracy:** ~0.95 (в зависимости от датасета и эпох обучения; измеряется на тестовых данных после обучения)
**Loss:** <0.1 на тесте (для оценки стабильности модели)

### Визуализация
В ноутбуке deepfake_detection.ipynb представлены визуализации:
- Графики точности и потерь по эпохам (для тренировки и валидации)
- Примеры изображений из датасета для предварительного просмотра
- Результаты предсказаний на тестовых изображениях с вероятностями классов (fake/real)
